| Type       | Description                                                | Examples                                            |
| ---------- | ---------------------------------------------------------- | --------------------------------------------------- |
| `feat`     | Introduces a new feature or functionality.                 | `feat: add dark mode support`                       |
| `fix`      | Fixes a bug or issue.                                      | `fix: correct typo in function name`                |
| `style`    | Code formatting, without changing logic (non-CSS related). | `style: fix code indentation`                       |
| `refactor` | Code refactoring without adding features or fixing bugs.   | `refactor: improve API structure for readability`   |
| `perf`     | Performance improvements.                                  | `perf: optimize loop for faster processing`         |
| `deps`     | Add, update, or remove libraries or dependencies.          | `deps: add Prettier and ESLint`                     |
| `config`   | Changes to project configuration files.                    | `config: tweak eslint rules`                        |
| `chore`    | Maintenance tasks (non-user facing).                       | `chore: update package-lock.json`                   |
| `docs`     | Documentation updates.                                     | `docs: add usage examples to README`                |
| `build`    | Changes to build scripts or CI/CD configurations.          | `build: update Dockerfile to use new base image`    |
| `test`     | Adding or updating tests.                                  | `test: add unit tests for user validation function` |
